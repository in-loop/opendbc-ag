# opendbc-ag

A MIT-licensed repository of CAN DBC files for agricultural equipment, covering ISO 11783 (ISOBUS) public summaries and the SAE J1939 ag-relevant subset, plus the Python tooling that extracts and validates them.

**Status:** `v0.1.0` — initial public release; see [`CHANGELOG.md`](CHANGELOG.md).

## What it does

opendbc-ag is the agricultural analog to comma.ai's `opendbc`: a community-maintainable, MIT-licensed corpus of CAN DBC files plus extraction, coverage, and scope-policy tooling. It does cover pure-standard PGNs only — ISO 11783 (ISOBUS) public summaries and the SAE J1939 ag-relevant subset. It does not cover reverse-engineered proprietary OEM PGNs.

Proprietary OEM PGNs (PGN IDs in `0xEF00` / `0x1EF00` / `0xFF00–0xFFFF` / `0x1FF00–0x1FFFF` ranges, or anything named `Proprietary*`/`Reserved*`) are out of scope and CI-rejected. (No existing MIT-licensed ag-CAN DBC corpus is comparable to comma.ai's `opendbc`; the commercial CSS Electronics ISOBUS DBC is closed and non-redistributable, and community projects each maintain partial DBCs without a shared baseline.)

## Quickstart

```bash
git clone https://github.com/in-loop/opendbc-ag.git
cd opendbc-ag

# Linux / macOS
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"

# Windows
python -m venv .venv
.venv\Scripts\activate
pip install -e ".[dev]"
```

### Verify

```bash
pytest opendbc_ag/tests/                              # expect 76+ passing
python -m opendbc_ag.tools.coverage_report            # regenerates docs/coverage.md
python -c "import cantools; print(cantools.database.load_file('opendbc_ag/dbc/iso11783_from_agisostack.dbc'))"
```

Next: see [`docs/cabana-for-ag-setup.md`](docs/cabana-for-ag-setup.md) for the end-to-end CAN sniffing + signal discovery workflow.

## How it works

- `opendbc_ag/dbc/` holds the three shipped DBC files; the extraction tools regenerate them from their public sources.
- `opendbc_ag/tools/` contains the extractors (`extract_agisostack.py`, `extract_vdma.py`, `extract_j1939_ag.py`), the `coverage_report.py` generator, the `dbc_inferrer.py` and `anonymize.py` helpers, and shared `_dbc_utils.py`.
- A scope-policy gate rejects proprietary/reserved PGN ranges so only pure-standard content lands.
- CI auto-regenerates `docs/coverage.md` on DBC changes.
- See [`docs/architecture.md`](docs/architecture.md) for the directory tour, data flow, and CI gates.
