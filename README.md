# opendbc-ag

> OSS canonical agricultural-CAN DBC repository — ISO 11783 (ISOBUS) + J1939 ag-relevant subset, MIT-licensed.

**Status:** `v0.1.0-private` — under maintainer review before public release.

## What

opendbc-ag is the agricultural analog to comma.ai's `opendbc` — a community-maintainable, MIT-licensed repository of CAN DBC files for agricultural equipment. Scope is **pure-standard PGNs only**: ISO 11783 (ISOBUS) public summaries and the SAE J1939 ag-relevant subset. No reverse-engineered proprietary OEM PGNs.

## Why

There is no widely-shared MIT-licensed ag-CAN DBC corpus comparable to comma.ai's `opendbc`. The commercial incumbent (CSS Electronics ISOBUS DBC) is closed and non-redistributable; community projects (AgOpenGPS, ISOBlue/Purdue, individual researcher repos) each maintain partial DBCs without a shared canonical baseline. opendbc-ag fills that gap with MIT-licensed coverage anyone can fork, modify, and ship.

## Install

```bash
git clone <repo-url>
cd opendbc-ag

# Linux / macOS
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"

# Windows
python -m venv .venv
.venv\Scripts\activate
pip install -e ".[dev]"
```

### Verify

```bash
pytest opendbc_ag/tests/                              # expect 76+ passing
python -m opendbc_ag.tools.coverage_report            # regenerates docs/coverage.md
python -c "import cantools; print(cantools.database.load_file('opendbc_ag/dbc/iso11783_from_agisostack.dbc'))"
```

Next: see [`docs/cabana-for-ag-setup.md`](docs/cabana-for-ag-setup.md) for the end-to-end CAN sniffing + signal discovery workflow.

## Contributing

See [`CONTRIBUTING.md`](CONTRIBUTING.md). PRs welcome for additional public-spec-derived content. **Proprietary OEM PGNs (PGN IDs in 0xEF00 / 0x1EF00 / 0xFF00–0xFFFF / 0x1FF00–0x1FFFF ranges, or anything named `Proprietary*`/`Reserved*`) are out of scope and CI-rejected.**
